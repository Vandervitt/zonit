# 落地页全流程 · 产品说明书

> 简版产品/项目说明，面向产品、运营与新成员快速理解。
> 工程实现细节（数据表、API、路由、门禁等）见详细设计：`docs/superpowers/specs/2026-06-17-landing-page-flow-design.md`。

## 一、这是什么

在 PULSAR 应用内，让登录用户**自助创建一张海外获客落地页并对外发布**的完整流程。用户从挑选行业模板开始，在编辑器里调整文案与图片，预览效果，最后一键发布到公开链接对外投放。

落地页定位：海外获客（leadgen）、**非交易**——引导访客通过 WhatsApp / Telegram / 表单 / 预约 / 咨询留资，不含下单、结账、订阅等交易环节。

### 本产品的三端

| 端 | 面向 | 职责 | 现状 |
|---|---|---|---|
| **官网**（marketing）| 潜在用户 | 介绍产品、吸引并转化为注册用户；漏斗前门 | 待补（目前仅 `/pricing` 公开）|
| **admin**（租户后台）| 租户（付费用户）| 租户自管落地页、数据、域名等——**本流程所在地**（PULSAR 仪表盘 `(dashboard)`）| 基本就绪 |
| **super-admin**（平台后台）| 平台运营 | 管理租户、模板、平台设置（`/admin`，`SUPER_ADMIN` 角色）| 已有雏形 |

角色体系：`USER`（租户）/ `ADMIN` / `SUPER_ADMIN`（平台）。

### 完整入口漏斗

**访问官网 → 跳转 admin → 注册/登录 → 进入落地页流程**。即：潜在用户在官网了解产品并点击 CTA → 进入 admin 端的注册/登录（复用现有应用认证）→ 登录后落到租户后台首页 → 走下文「核心流程」。super-admin 在后台维护模板库与租户，模板供 admin 端选用。

## 二、谁用、解决什么

- **使用者**：PULSAR 的登录用户（各行业获客方），主要是面向个人创业者和小型企业。
- **价值**：无需开发，选模板 → 改内容 → 发布，几分钟内上线一张可投放的落地页。

## 三、核心流程（七步）

| 步骤 | 用户做什么 | 看到/得到什么 |
|---|---|---|
| ⓪ 访问官网 | 在官网了解产品、点击 CTA | 跳转到 admin 端注册/登录 |
| ① 注册/登录 | 用应用账号注册或登录（复用现有认证）| 进入租户后台首页（仪表盘）|
| ② 进入「落地页」 | 点侧边栏「落地页」 | 自己的落地页列表（草稿/已发布）|
| ③ 新建 + 选模板 | 点「新建」，从模板库挑一套行业模板 | 自动创建页面并进入编辑器 |
| ④ 编辑 | 修改各区块的文案、图片、按钮等 | **自动保存**，顶栏显示保存状态 |
| ⑤ 预览 | 右栏实时预览；或点「全屏预览」 | 手机/桌面下所见即发布后效果 |
| ⑥ 发布 | 点「发布」，确认页面路径 | 发布到**用户自己的域名**下，得到对外公开链接（详见第六节发布域名约束）|
| ⑦ 管理 | 在列表里再编辑 / 取消发布 / 删除 | 闭环回到应用内管理 |

一句话流程：**官网 → 注册/登录 → 选模板 → 编辑 → 预览 → 发布(自有域名) → 管理**。

## 四、主要功能点

- **模板库**：内置多套行业模板（护肤、牙科、家装太阳能、美妆护肤等），开箱即用、可自由调整。
- **可视化编辑器**：12 种内容区块（首屏、数据、套餐、产品、前后对比、流程、信任、特性、评价、故事、倒计时、常见问题、安全保障、页脚、悬浮按钮等），可增删、排序、改文案换图。
- **自动保存**：编辑即存，无需手动点保存。
- **双重预览**：编辑器内实时预览（手机/桌面取景）+ 全屏预览整页效果。
- **一键发布**：自动生成可读链接地址，可手动改并查重；发布前校验页面结构完整性。
- **发布管理**：再编辑、取消发布（链接保留）、删除，全部在应用内完成。

## 五、本期范围

**做**：上述登录 → 选模板 → 编辑 → 预览 → 发布 → 管理的完整闭环，并在侧边栏新增「落地页」入口。其中**自有域名绑定/验证 + 发布到自有域名（按 host 的多租户渲染）纳入本期**——复用现有 Domains/Vercel 能力，不新造域名管理 UI（详见第六节发布域名约束）。

**暂不做（后续迭代）**：数据统计/像素、SEO 编辑面板、版本历史、可分享的草稿预览链接、按套餐加水印、多语言、多人协作。

## 六、关键约束

- **发布域名约束（重要）**：落地页**只能发布到用户自己的域名下**——用户须先绑定并验证自有域名，平台**不提供公共子路径 / 共享子域名托管**；**未绑定自有域名则无法发布**。这保证每张落地页都跑在用户自己的品牌域名上，利于品牌、SEO、广告投放与可信度。
- 落地页**不引入任何交易语义**（下单/结账/购物车/订单/订阅/退款/货到付款等），转化只走咨询/预约/留资。
- 发布后的公开页面对外可访问；编辑与预览需登录。
- 站点数据按用户隔离，仅本人可见可改。

## 七、功能扩展 Idea 地图

> 下列为产品后续可演进方向，**非本期范围**（本期仍以第五节的 happy-path 闭环为准）。一条红线贯穿始终：落地页保持**非交易**，所有方向都围绕「获客 / 留资 / 投放」，不碰下单结账。
>
> 背景：「多方 pixel」并非从零——`templates/template` 的源数据 `beauty-landing-page.ts` 里已有 `pageMeta.analytics.pixels`（meta + `page_view→Lead`/`cta_click→WhatsAppClick`/`form_submit→FormSubmit` 事件映射），只是当前 `LandingPageDraft` 精简时去掉；SEO 字段同理。这些都可在 schema 中复活并扩展。

### 1. 追踪与多方 Pixel
- **多平台像素注入**：Meta/Instagram、Google（GA4 + Google Ads）、TikTok/抖音、小红书、Snapchat、Pinterest、LinkedIn、X、Bing、Kwai——编辑器填 pixel ID 即注入，无需改代码。
- **统一事件映射**：页面行为映射到各平台转化事件（`page_view→Lead`、`cta_click→Contact`、`form_submit→SubmitApplication`、`whatsapp_click`），一处配置、多平台上报。
- **服务端转化 API（高价值）**：Meta CAPI、TikTok Events API、Google Enhanced Conversions，绕过 iOS/广告拦截，回传更准，直接提升投放 ROAS。
- **UTM 捕获与透传**：自动抓 `utm_*`，带进 WhatsApp/Telegram 深链与表单提交做归因。
- **GTM 容器**：贴一个 Google Tag Manager 容器 ID，给进阶用户自托管标签。
- **同意管理（CMP）**：欧盟流量跑 pixel 必须有 Cookie 同意横幅——pixel 的合规前置。

### 2. 留资与线索管理（把「获客」做成闭环）
- **表单区块**：新增 `form` 区块类型、自定义字段，提交即线索。
- **线索收件箱**：仪表盘看/筛/导出线索（CSV），新线索邮件/WhatsApp 通知。
- **Webhook / Zapier / Make**：线索实时推到用户自己的 CRM（HubSpot、Notion、Google Sheet）。
- **点击咨询追踪**：WhatsApp/Telegram 点击计数与预填深链。

### 3. 转化优化
- **A/B 测试**：一页多变体、按流量分流、看哪版转化高。
- **退出挽留弹窗 / 滚动触发 CTA**、**实时社会证明**（「刚刚有 N 人发起咨询」）。

### 4. SEO 与表现
- **SEO 编辑面板**：把 `pageMeta.seo`（title/description/OG/canonical/robots）加回 schema。
- **JSON-LD 结构化数据**、**sitemap**、**图片优化/CDN**、Core Web Vitals 体检。

### 5. 域名与品牌
- **自定义域名**（打通现有 Domains 区块到 `/p/[slug]`）、**品牌套件**（主题色/字体/Logo 统一换肤，渲染器已有 theme 机制）、**按套餐去水印**。

### 6. 本地化（海外获客刚需）
- **多语言落地页 + AI 翻译**、**按地域切换语言**（纯展示，不涉及货币交易）、RTL 支持。

### 7. AI 内容（贴合「无需开发」定位）
- **AI 文案**：按行业一键生成/改写各区块（用最新 Claude 模型）。
- **AI 配图**、**一句话 / 一个 URL → 整页生成**、语气调整。

### 8. 模板与复用
- **更多行业模板 + 模板市场**、**另存为模板**、**一键克隆页面**、区块素材库。

### 9. 发布生命周期与协作
- **定时发布/下线**、**版本历史与回滚**、**密码保护/可分享草稿预览**、**团队工作区与角色/审批**、**二维码生成**（线下导流）、**link-in-bio 模式**。

### 10. 内置数据看板
- **漏斗分析**（曝光→CTA 点击→线索）、流量来源、设备分布——不出应用即看投放效果。

## 八、优先级建议（面向个人创业者 / 小企业 + 付费投放）

按「投放 ROI 直接相关 + 复用现有 schema/能力」排序：

1. **多方 Pixel + 服务端转化 API + UTM** —— 投放用户的命脉，旧 schema 已有雏形，落地最快、价值最高。
2. **表单区块 + 线索收件箱 + Webhook** —— 把「获客」做成可见可用的闭环（现仅 WhatsApp 跳转）。
3. **SEO 面板 + 自定义域名** —— 自然流量 + 品牌可信度，域名可复用现有 Domains 区块。
4. **AI 文案** —— 强化「几分钟上线」的核心卖点。
5. **A/B 测试** —— 转化提升，但工程复杂度较高，可稍后。

**合规提醒**：跑 pixel 基本要配 **Cookie 同意（CMP）**，尤其欧盟流量；建议与 pixel 同期做，否则有合规风险。
