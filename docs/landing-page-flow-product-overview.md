## 一、产品定位

Zap Bridge 落地页是 Zap Bridge 平台内的**自助式落地页搭建与发布工具**。用户从行业模板起步，在可视化编辑器中调整文案与图片，预览后**一键发布到自己的域名**，用于海外广告投放与自然流量获客。

- **品类**：落地页搭建器（Landing Page Builder）/ 营销获客工具。
- **场景**：海外获客（leadgen）。
- **转化方式（非交易）**：引导访客通过 WhatsApp / Telegram / 表单 / 预约 / 咨询**留资**，**不含**下单、结账、购物车、订单、订阅、退款等任何交易环节。

## 二、目标用户与核心诉求

- **主要用户**：做海外获客的**个人创业者与小型企业**（投手 / 独立卖家 / 本地服务商）。
- **要完成的任务（JTBD）**：「我要快速上线一张能跑广告、能收线索的落地页，最好不写代码、不求人。」
- **痛点**：建站门槛高、改文案要找开发、投放像素与归因难配、上线慢。
- **价值主张**：**选模板 → 改内容 → 发布**，几分钟上线；页面跑在自己的品牌域名上，利于品牌、SEO 与投放可信度。

## 三、产品目标与成功度量

| 目标 | 建议度量指标 | 示例目标（待校准）|
|---|---|---|
| 上手快 | 注册到首次发布的中位时长（Time-to-First-Publish）| < 15 分钟 |
| 能激活 | 注册后 7 天内成功发布 ≥1 张落地页的用户占比 | 持续提升 |
| 走完闭环 | 草稿 → 发布的转化率 | 持续提升 |
| 有黏性 | 发布用户的次月再编辑 / 再发布率 | 持续提升 |

> 指标口径与目标值在上线后结合数据校准；此处给出度量框架而非承诺数值。

## 四、三端架构与角色

本产品分三端，本说明书聚焦 **admin 端**的落地页流程：

| 端 | 面向 | 职责 | 现状 |
|---|---|---|---|
| **官网**（marketing）| 潜在用户 | 介绍产品、转化为注册用户（漏斗前门）| 待补：目前仅 `/pricing` 公开，无营销首页 |
| **admin**（租户后台）| 租户（付费用户）| 自助管理落地页、数据、域名——**本流程所在地**（Zap Bridge 仪表盘）| 基本就绪 |
| **super-admin**（平台后台）| 平台运营 | 管理租户、模板库、平台设置 | 已有雏形（`/admin`）|

- **角色**：`USER`（租户）/ `ADMIN` / `SUPER_ADMIN`（平台）。
- **职责边界**：super-admin 维护**模板库**与**租户**，模板供 admin 端选用；admin 端用户只能管理自己的页面与数据（按用户隔离）。

## 五、用户旅程（端到端漏斗）

**访问官网 → 跳转 admin → 注册/登录 → 选模板 → 编辑 → 预览 → 发布(自有域名) → 管理**

| 步骤 | 用户做什么 | 得到 / 看到 |
|---|---|---|
| ⓪ 访问官网 | 在官网了解产品、点击 CTA | 跳转到 admin 端注册/登录 |
| ① 注册 / 登录 | 用账号注册或登录（复用现有应用认证）| 进入租户后台首页 |
| ② 进入「落地页」 | 点侧边栏「落地页」 | 自己的落地页列表（草稿 / 已发布）|
| ③ 新建 + 选模板 | 「新建」→ 从模板库挑一套行业模板 | 自动创建页面并进入编辑器 |
| ④ 编辑 | 改各区块的文案、图片、按钮等 | **自动保存**，顶栏显示保存状态 |
| ⑤ 预览 | 右栏实时预览；或「全屏预览」 | 手机 / 桌面所见即发布后效果 |
| ⑥ 发布 | 选择已验证的**自有域名**并确认 | 页面发布到该域名根路径，得到对外链接（见第七节域名约束）|
| ⑦ 管理 | 在列表里再编辑 / 取消发布 / 删除 | 应用内闭环管理 |

## 六、核心功能

- **模板库**：内置多套行业模板（护肤、牙科、家装太阳能、美妆护肤等），开箱即用、可自由调整。
- **可视化编辑器**：**12 种可排序内容区块**（数据展示、套餐、产品、前后对比、服务流程、信任、特性、评价、产品故事、倒计时、常见问题、安全保障）+ **3 个页面级固定件**（首屏 Hero、页脚 Footer、悬浮按钮）；区块可增删、排序、改文案换图。
- **自动保存**：编辑即存，无需手动保存。
- **双重预览**：编辑器内实时预览（手机 / 桌面取景）+ 全屏预览整页效果。
- **自有域名发布**：发布前选择一个已验证的自有域名，页面上线到该域名；发布前校验页面结构完整性。
- **发布管理**：再编辑、取消发布（保留域名绑定）、删除，全部在应用内完成。

## 七、本期范围

**做（In Scope）**

- admin 端完整闭环：注册/登录 → 选模板 → 编辑（自动保存）→ 预览（实时 + 全屏）→ 发布 → 管理，并在侧边栏新增「落地页」入口。
- **自有域名绑定 / 验证 + 发布到自有域名（按 host 的多租户渲染）**——复用现有 Domains / Vercel 能力，不新造域名管理 UI。

**不做（Non-Goals，留后续迭代）**

- 官网营销站建设（独立 surface，本期只承接其注册/登录落点）。
- super-admin 在线管理「新流程模板库」（本期模板用内置模板）。
- 数据统计 / 投放像素、SEO 编辑面板、版本历史、可分享草稿预览、按套餐加水印、多语言、多人协作。

## 八、关键约束与合规

**硬约束（不可违背）**

- **自有域名发布**：落地页**只能发布到用户自己的、已绑定并验证的域名下**；平台**不提供公共子路径 / 共享子域名托管**；**未绑定自有域名则无法发布**。目的：每张页面跑在用户品牌域名上，利于品牌、SEO、投放可信度。
- **非交易**：落地页**不引入任何交易语义**（下单 / 结账 / 购物车 / 订单 / 订阅 / 退款 / 货到付款等），转化只走咨询 / 预约 / 留资。

**访问与数据**

- 已发布页面对外公开（在用户自有域名上）；编辑与预览需登录。
- 数据按用户隔离，仅本人可见可改。

## 九、产品路线图（非本期，按优先级）

> 演进方向，均**非本期范围**。贯穿红线：保持**非交易**，所有方向围绕「获客 / 留资 / 投放」。
>
> 背景：「多方 pixel」与「SEO」并非从零——源模板 `templates/template/data/beauty-landing-page.ts` 中已有 `pageMeta.analytics.pixels` 与 `pageMeta.seo` 的结构，当前 `LandingPageDraft` 精简时去掉，可在 schema 中复活并扩展。

**推荐优先级（面向个人创业者 / 小企业 + 付费投放）**

| 优先级 | 方向 | 理由 |
|---|---|---|
| P0 | 多方 Pixel + 服务端转化 API + UTM | 投放用户的命脉，旧 schema 已有雏形，落地快、价值高 |
| P1 | 表单区块 + 线索收件箱 + Webhook | 把「获客」做成可见可用闭环（现仅 WhatsApp 跳转）|
| P2 | SEO 编辑面板 | 自然流量与可被搜索引擎正确收录 |
| P3 | AI 文案 | 强化「几分钟上线」的核心卖点 |
| P4 | A/B 测试 | 转化提升，但工程复杂度较高 |

**完整方向清单**

1. **追踪与多方 Pixel**：多平台像素注入（Meta/Instagram、Google GA4 + Ads、TikTok/抖音、小红书、Snapchat、Pinterest、LinkedIn、X、Bing、Kwai，填 ID 即注入）；统一事件映射（`page_view→Lead`、`cta_click→Contact`、`form_submit`、`whatsapp_click`）；服务端转化 API（Meta CAPI、TikTok Events API、Google Enhanced Conversions，绕过 iOS/拦截、提升 ROAS）；UTM 捕获与透传；GTM 容器。
2. **留资与线索管理**：表单区块；线索收件箱（看/筛/导出/通知）；Webhook / Zapier / Make 推送至用户 CRM；点击咨询追踪与预填深链。
3. **转化优化**：A/B 测试；退出挽留弹窗 / 滚动触发 CTA；实时社会证明。
4. **SEO 与表现**：SEO 编辑面板（title/description/OG/canonical/robots）；JSON-LD；sitemap；图片优化 / CDN；Core Web Vitals。
5. **品牌**：品牌套件（主题色 / 字体 / Logo 统一换肤）；按套餐去水印。
6. **本地化**：多语言落地页 + AI 翻译；按地域切换语言（纯展示）；RTL。
7. **AI 内容**：AI 文案（按行业生成 / 改写）；AI 配图；一句话 / 一个 URL → 整页生成；语气调整。
8. **模板与复用**：更多行业模板；模板市场；另存为模板；一键克隆；区块素材库。
9. **发布生命周期与协作**：定时发布 / 下线；版本历史与回滚；密码保护 / 可分享草稿预览；团队工作区与角色 / 审批；二维码生成；link-in-bio。
10. **内置数据看板**：漏斗分析（曝光 → CTA 点击 → 线索）、流量来源、设备分布。

**合规提醒**：跑 pixel 基本要配 **Cookie 同意（CMP）**，尤其欧盟流量；建议与 P0 像素同期落地，否则有合规风险。

## 十、术语表

| 术语 | 含义 |
|---|---|
| 租户（Tenant）| admin 端的付费用户，对应角色 `USER`；只能管理自己的页面与数据 |
| 落地页（Landing Page）| 用户创建、可发布的单页；内容由 `LandingPageDraft` schema 描述 |
| 区块（Section / Block）| 落地页的内容单元；12 种可排序区块 + 首屏 / 页脚 / 悬浮按钮 3 个固定件 |
| 模板（Template）| 预置的整页内容样例，供新建时选用（内置多套行业模板）|
| 自有域名（Custom Domain）| 用户绑定并验证后用于发布的域名；发布的唯一落点 |
| 多租户渲染（Multi-tenant）| 按访问的域名（host）解析到对应已发布页面并渲染 |
| 草稿 / 已发布（Draft / Published）| 页面状态；仅已发布页可在自有域名对外访问 |
