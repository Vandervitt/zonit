# Coding Style

- React components use PascalCase.
- Events use camelCase.
- Use clear, descriptive names with consistent casing.
- Middleware file is `proxy.ts`.
- Prefer minimal, readable code over elaborate abstractions.
- Break complex tasks into small, testable units.
- MVP focus: least code change, happy path first.
- Limit guards and fallbacks to what production has proven necessary.
- Remove dead, unreachable, or unused code.
- Do not write multiple versions or backups for the same logic.
- Before writing Next.js code, read the relevant guide in `node_modules/next/dist/docs/`.
