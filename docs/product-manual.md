## 一、产品定位

Zap Bridge 是一个 **SaaS 落地页搭建与获客工具**（Landing Page Builder / lead-gen 工具），聚焦**海外获客（leadgen）**场景：

- **品类**：自助式落地页搭建器 + 投放追踪 + 线索管理。
- **转化方式（核心红线：非交易）**：生成的落地页只引导访客通过 **WhatsApp / Telegram / 表单 / 电话 / 邮件 / 预约咨询**留资，**绝不包含**下单、结账、购物车、订单、订阅、退款、货到付款等任何电商交易环节。平台自身向 SaaS 客户收费（订阅套餐，Lemon Squeezy 计费），但平台收费与生成页面的能力边界严格分离。
- **价值主张**：**选模板（或 AI 生成）→ 改内容 → 发布到自有域名**，几分钟上线；页面跑在用户自己的品牌域名上，利于品牌、SEO 与投放可信度；像素/CAPI/线索链路开箱即用。

### 目标用户与 JTBD

- **主要用户**：做海外获客的个人创业者与小型企业（广告投手、独立卖家、本地服务商、小型 Agency）。
- **要完成的任务**：「我要快速上线一张能跑广告、能收线索的落地页，不写代码、不求人；像素归因要配得上，线索要收得到。」
- **痛点**：建站门槛高、改文案要找开发、投放像素与归因难配、上线慢、同模板页面被广告平台判重（同质化风险）。

## 二、三端架构与角色

| 端 | 路由 | 面向 | 职责 |
|---|---|---|---|
| **官网**（marketing）| `/`、`/pricing`、`/anti-ban` | 潜在用户 | 营销首页、定价页、反同质化叙事页；转化为注册用户 |
| **admin**（租户后台）| `/admin/*` | 租户（`USER`）| 落地页、线索、分析、域名、素材、账单的自助管理——产品主体 |
| **super-admin**（平台后台）| `/super-admin/*` | 平台运营（`SUPER_ADMIN`）| 管理租户/用户、平台设置 |

- 认证：NextAuth（邮箱注册 + Google 登录）；数据按用户隔离，仅本人可见可改。
- 公开面：已发布落地页在用户自有域名对外公开（按 host 多租户渲染）；预览路径 `/p/[slug]`；编辑与后台需登录；未知路由渲染 404。

## 三、用户旅程（端到端）

**访问官网 → 注册/登录 → 建页（模板 / AI 一键生成）→ 编辑（自动保存）→ 预览 → 绑定并验证自有域名 → 发布 → 广告投放（像素 + CAPI 归因）→ 访客留资 → 线索收件箱 / Webhook 推 CRM → 数据看板复盘**

| 步骤 | 用户做什么 | 产品提供什么 |
|---|---|---|
| ① 建页 | 「新建」→ 选行业模板，或 AI 一句话生成整页 | 30+ 行业模板；AI 生成含文案 + Unsplash 自动配图 |
| ② 编辑 | 可视化调整各区块文案、图片、CTA | 自动保存；撤销/重做（Cmd+Z，50 步，AI 生成可一步撤销）；区块增删排序；MediaPicker（Unsplash + 上传）|
| ③ 预览 | 编辑器内手机/桌面实时预览 + 全屏预览 | 所见即发布后效果；草稿可分享预览链接 |
| ④ 发布 | 选择已验证的自有域名并确认 | 发布前校验（结构完整性、WhatsApp 占位号拦截等，问题项可点击定位）；快照制发布——发布后编辑只改草稿、线上不变，「更新发布」才切新版 |
| ⑤ 投放 | 在 Meta/TikTok/Google 后台配广告 | 客户端像素 + 服务端 CAPI 回传 + UTM 归因 |
| ⑥ 收线索 | 访客提交表单 / 点 WhatsApp | 线索收件箱；Webhook 实时推送 CRM（含签名与重试）|
| ⑦ 复盘 | 看数据 | 分析看板：曝光 → CTA 点击 → 线索的转化漏斗 |

## 四、核心功能

### 1. 模板库与 AI 建页

- **30+ 行业模板**（第一梯队电商种草留资：6 行业 × 5 子品类共 29 套，另有 solar/dental 等线索类范式），全程非交易、WhatsApp 留资范式，高风险品类强制 disclaimer。四档套餐**全量开放模板**。
- **AI 一键生成整页**：按行业/描述生成文案与结构，自动 Unsplash 配图（含前后对比、头像等素材对），多 LLM 适配（Qwen/Gemini/DeepSeek/OpenRouter）。AI 生成/改写按套餐计额度。

### 2. 可视化编辑器

- **12 种可排序内容区块**：数据展示、套餐（Offer，纯展示文案无账单语义）、产品、前后对比、服务流程、信任、特性、评价、产品故事、倒计时、常见问题、安全保障；另有 **LeadForm 留资表单区块**。
- **3 个页面级固定件**：首屏 Hero、页脚 Footer（必含合规链接）、悬浮按钮。
- 自动保存、双重预览（编辑器内实时 + 全屏）、SEO 编辑面板（title/description/OG 等 + JSON-LD 派生）、品牌主题（6 套主题色）。

### 3. 发布与自有域名

- **只能发布到用户自己已绑定并验证的域名**（平台不提供公共子路径/共享子域名托管；Free 档无域名额度，可在线预览但发布需升级）。
- **不支持中国大陆管辖域名**：添加域名时拦截 `.cn` 全系（含 `.com.cn`/`.gov.cn` 等二级）及大陆 IDN 顶级域（`.中国`/`.中國`/`.公司`/`.网络`，按 punycode 判断），此类域名受备案与注册局政策影响（实名核验、serverHold 暂停解析），不适合发布出海落地页；前端表单即时提示 + API 兜底（错误码 `domain_tld_blocked`）。
- **一域名一页**：每个域名同一时间只承载一张已发布页面（域名根路径），重复发布为覆盖式替换（原页面下线、旧投放链接失效）；反向支持一页复制多份发布到多个域名。设计动机是投放风控——广告平台封禁与信誉以域名为单位，一域名一 offer 隔离连坐风险。
- 多注册商 DNS 指引、按 host 的多租户渲染、发布前校验、再编辑（快照制：只改草稿，「更新发布」才上线）/取消发布/删除全应用内闭环。

### 4. 投放追踪与归因

- **基础像素**（四档全含，体验漏斗）：1× Meta Pixel 客户端追踪。
- **多平台追踪 + CAPI**（Pro/Agency）：Meta / TikTok / Google 客户端追踪 + Meta / TikTok 服务端转化回传（落库入队 + 即时 flush，cron 每日兜底），绕过 iOS/广告拦截提升 ROAS。
- 事件口径全部 lead-gen 化：`Lead / Contact / FormSubmit / WhatsAppClick`，无任何 purchase/checkout 事件。
- **合规**：欧盟/EEA/GB 访客 CMP Cookie 同意门控，第一方埋点随同意状态放行。

### 5. 线索管理

- **线索收件箱**（`/admin/leads`）：查看、筛选留资线索。
- **线索 Webhook**（Pro/Agency）：新线索实时 POST 到用户 CRM / Zapier / Make，含签名与失败重试。
- 归因：识别线索来自哪个 CTA / 表单 / 渠道。

### 6. 反同质化（Agency 专属）

- 更换页面**变体种子**打散同模板页面的结构指纹，降低被广告平台判重的概率（8 组 salt，明确不做 cloaking）。官网有专门叙事页 `/anti-ban`。对外统一命名「反同质化」。

### 7. 分析看板

- 投放分析页：曝光 → CTA 点击 → 线索的**转化漏斗**、线索统计、流量数据。

## 六、关键约束（硬红线）

1. **非交易**：生成页面的 schema、文案、组件、SEO、埋点事件、模板一律不得引入交易语义（无 PaymentTrust/ShippingInfo 区块，Offer 的 `priceText` 仅为展示文案）。
2. **自有域名发布**：未绑定验证自有域名不能发布，无公共托管路径。
3. **线索有效性**：`primaryConversion` 必填且指向真实留资路径（URL/电话/邮箱/表单）；lead form 至少收集一个可触达联系字段（phone/email/whatsapp/telegram），仅 name 不算线索；每页至多一个 leadForm。
4. **合规结构**：页脚必含合规链接；高风险品类（健康/美容/金融等）支持 disclaimer、避免保证性承诺；欧盟流量走 CMP 同意。
5. **工程**：Tailwind only（无自定义 CSS/内联样式）；`proxy.ts` 做中间件；中文 Conventional Commits。

## 七、技术形态（简）

- **栈**：Next.js（App Router，部署 Vercel）+ Postgres（Neon，手动 SQL 迁移 node-pg-migrate）+ Vercel Blob（素材）+ NextAuth + Ant Design（后台）/ Tailwind + Radix（编辑器与公开页）+ Lemon Squeezy（计费）+ Sentry / Speed Insights（监控）。
- **代码分区**：`app/`（三端路由 + API）、`landing-editor/`（编辑器与模板样例）、`landing-renderer/`（公开页渲染、变体、水印、追踪）、`lib/`（plans/tracking/capi/leads/proxy 等域逻辑）、`types/schema.ts` + `lib/schema.zod.ts`（落地页 schema 单一事实源）。
- **平台约束**：Vercel Hobby 计划 cron 最快每日一次——异步投递（CAPI/Webhook）happy-path 靠落库后 `after()` 即时 flush，cron 仅兜底。
