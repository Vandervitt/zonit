# UI 一致性治理（现状收口）

分支：`refactor_20260713_UI一致性治理`　日期：2026-07-13

## 背景与体检结论

项目现状是「两套设计系统按区域划分 + 几处治理漏洞」，底子健康，无需推倒重来：

- **令牌源**：Tailwind v4 CSS-first，`styles/theme.css` 为「全局主题唯一来源」，语义 token 完整。
- **前台**（官网 / 落地页）：shadcn/ui（Radix + Tailwind）47 组件 + `lib/theme.ts` 成套类名预设，已收口。
- **后台**（admin / super-admin）：Ant Design v5，经 `lib/theme/antd-theme.ts` 的 ConfigProvider 注入。

不一致的真实来源：
1. 两系统品牌色 `#0e9fe4` 在 theme.css 与 antd-theme.ts **各写一份手动同步**，靠注释约束 → 会漂移。
2. admin/super-admin 有 7 处 `#0e9fe4` 直接写进 inline `style`，绕过 token。
3. MUI v7 + emotion **全仓 0 引用**，死依赖。
4. 无 lint/测试阻止硬编码色、内联样式回流。

### 对已批方案的一处修正

原方案「ESLint 全仓禁 hex 字面量 error + 白名单极少数」在实现前侦察发现：全仓 tsx 有 ~49 处**合法** hex（Google/Microsoft 登录 logo SVG、Recharts 图表色、语义状态色），一刀切会搞崩构建。故护栏收窄为**只禁品牌主色字面量 `#0e9fe4`**（brand.ts 除外）+ 禁 `@mui/*`/`@emotion/*` import，精准打击根因、不误伤。
