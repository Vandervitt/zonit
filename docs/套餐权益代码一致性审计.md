# 套餐权益 · 代码一致性审计

日期：2026-07-15　状态：**只记录，暂不改动**（等用户逐项拍板）

## 目的

核对每项套餐权益「声明值(`lib/plans.ts`) ↔ 页面文案 ↔ **实际产生限制的执行代码**」三者是否一致。
重点不是文案与 `plans.ts` 对不对，而是**有没有真正的执行代码在按声明的数值/开关做限制**。

## 详细证据

### ✅ 真实生效的限额（执行点）

- **落地页数量**：`app/api/landing-pages/route.ts:33`、`generate/route.ts:41`、`[id]/duplicate/route.ts:15` —— 均读 `PLANS[plan].landingPagesLimit`，`existing.length >= limit` → 403。
- **域名**：`app/api/domains/route.ts:41,51,70` 与 `[id]/route.ts:38` —— 服务端读 `PLANS[plan].domainsLimit`，启用/新增前 `count >= limit` → 403（不只客户端页面）。
- **AI 次数**：`lib/ai/usage.ts` 的 `hasAllowance`/`checkAndConsume` —— `monthCount` 用 `date_trunc('month', now())` 统计 `ai_usage`，`quota===Infinity` 放行；page 超额可用 `ai_credit_balance`。数值由 `lib/plans.test.ts` 锁定（3/15/80/300、10/100/∞/∞）。

### ⚠️ 声明了但没执行 / 没实现

- **模板 `allTemplates`**：仅 `UpgradeDialog.tsx:70` 用于**展示**一句升级卖点；`landing-editor` 的 `TemplateGallery`/`registry`/`templateFilter` 无任何 plan 过滤 → 全档位可用全部模板。
- **水印 `hasWatermark`**：全仓仅 `billing/page.tsx:98`(展示) 和 `UpgradeDialog.tsx:62`(卖点) 引用；`landing-renderer` **无任何水印渲染** → 免费页也没有水印，"去除水印"无实体。
- **高级追踪 `advancedTracking`**：仅 `plans.ts` + 对比表行引用；追踪/像素相关文件（`app/p/[slug]`、`landing-editor` 等）**无 plan 门控** → free/starter 同样能配 TikTok/CAPI。
- **反同质化风控 `antiBan`**：除 plans/营销文案外**零引用**，无实现。
- **AI 多语言翻译 `aiTranslation`**：无 translate 路由/函数，无实现。

## 文案层面的连带问题（页面在暗示不存在的限制）

- 对比表「去除品牌水印」free/starter 显示 ✗（=有水印）—— 实际无人有水印。
- 对比表「全矩阵像素追踪 (TikTok/CAPI)」free/starter 显示 ✗ —— 实际人人可用。
- 首页 Starter 卖点「5 款王牌爆款模板」、升级弹窗「解锁全部 15+ 行业模板」—— 暗示模板受限，实际不限。
- pro/agency 卖点「反同质化风控引擎」、agency「AI 多语言翻译」—— 宣传了未实现的功能。
